# スキル: 仮説抽出（感情表現 → 検証可能仮説の変換）

## 目的

`draft.md` の自由形式テキストから「データで検証可能な仮説」を抽出する。
感情・印象表現を背後にある「事実仮説」に読み替え、FOOTBALL LAB 等のデータで検証できる形式に変換する。

---

## 変換ルール

### 基本原則

- 感情・印象表現 → 背後にある「事実仮説」に読み替える
- 期待・願望・お願いは除外する（検証不可能なため）
- 1つの原文から複数の仮説が生まれてもよい

### 変換例

| 原文（感情・印象表現） | 変換後（検証可能仮説） |
|---|---|
| 「プレッシャーが全く効かなくなった」 | 「後半、大分のプレス成功率・奪取CBPが低下した」 |
| 「前半30分くらいまではトリニータのペース」 | 「前半序盤は大分のポゼッション・攻撃CBPが優位だった」 |
| 「赤い選手がわんさかいてトリニータの選手がまるでいなかった」 | 「後半、大分の自陣への侵入・守備CBPが悪化した」 |
| 「相手の左サイドで崩され出した」 | 「相手の左サイドからの攻撃（攻撃CBP）が際立っていた」 |
| 「押し込まれて悪循環だった」 | 「後半、大分のポゼッション・守備CBPが低下した」 |
| 「横綱相撲してたら土俵際まで追い詰められた」 | 「前半途中から優位な状況を逆転された（ポゼッション・CBP推移）」 |

---

## 除外ルール（スキップする表現）

以下のパターンは仮説化せずスキップする:

| パターン | 例 | 理由 |
|---|---|---|
| 期待・願望 | 「後半は立て直してくれよ」「〜に期待する」 | 検証不可能 |
| 純粋な感情 | 「つらい」「悔しい」「うれしい」 | 事実仮説なし |
| 無根拠な推測 | 「なんとなく」「気のせいかもしれないが」 | 検証の根拠なし |
| 監督・選手への個人的評価 | 「ヨモさんの立て直しに期待する」 | 主観的評価 |

---

## 出力形式

```markdown
## 抽出された仮説リスト

1. 仮説: [仮説の内容]
   根拠となった原文: 「[draft.md からの引用]」

2. 仮説: [仮説の内容]
   根拠となった原文: 「[draft.md からの引用]」
```

---

## 補足

- X（Twitter）等からのコピペ引用も同様に処理する
- 試合情報（日付・スコア・対戦相手・会場）は仮説ではなく「前提情報」として別途記録する
- 前半・後半などの時間帯が明示されている場合はそれを仮説に含める
